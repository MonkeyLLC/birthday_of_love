# 扭蛋机改造设计文档

## 1. 背景

当前项目的抽奖页已经具备完整的基础能力：

- 有独立的扭蛋机区域
- 有 GSAP 入场、待机、抽奖、结果弹窗动画
- 有奖品池、抽奖记录、扭蛋币和签到联动

现在的问题不在于“能不能抽”，而在于“扭蛋感还不够强”：

- 扭蛋机外观偏静态，像一个可爱的插画组件，但不像真实会运作的机器
- 机内球体过于统一，缺少层次、材质变化和随机感
- 当前流程是“点击抽奖 -> 出奖 -> 弹窗”，仪式感还不够
- 抽奖结果缺少“先拿到扭蛋，再亲手打开”的参与感

本次改造的目标，是参考 Gashapon 网站的核心优点，但不照搬它的整页全屏工具站形态。

## 2. 参考拆解结论

参考站值得借鉴的，不是它的页面布局，而是它的开奖节奏：

1. 用户不是点击一次就直接看到结果
2. 页面会先让用户感受到“机器正在出一个具体的球”
3. 球本身会成为视觉中心
4. 用户还需要再进行一次“打开球”的动作
5. 结果在打开之后才正式揭晓

这套逻辑让抽奖更像“拿到一个真正的扭蛋”，而不是普通按钮抽卡。

本项目的取舍如下：

- 保留现在的卡片式扭蛋机布局，不改成全屏沉浸式页面
- 保留目前的少女感、生日礼物感、恋爱氛围
- 借鉴参考站的球体表现、开奖阶段、点击开蛋的交互节奏
- 不借鉴参考站偏工具网站的功能型按钮布局

## 3. 改造目标

本次扭蛋机改造要同时满足 4 个目标：

1. 让扭蛋机“更像机器”
2. 让机内球体“更像真实扭蛋”
3. 让抽奖流程“更有阶段感和期待感”
4. 让整体视觉仍然符合本项目的可爱、少女、表白主题

## 4. 设计方向选择

本次有三个可选方向：

### 方案 A：轻量视觉升级

只调整机身和球体样式，保留现有抽奖流程。

优点：

- 改动小
- 风险低
- 很快可以完成

缺点：

- 交互惊喜感提升有限
- 很难真正接近参考站那种“扭蛋体验”

### 方案 B：卡片式扭蛋机 + 分阶段开蛋体验

保留现在页面结构，但重做机身细节、球体层次和开奖流程，在机器内部和机器前景里完成“出蛋 -> 点蛋 -> 开蛋 -> 揭晓”。

优点：

- 最符合当前项目结构
- 能明显提升互动感
- 手机端表现稳定
- 既能参考 Gashapon，又不会变成工具网站

缺点：

- 需要重构抽奖阶段逻辑
- 动画编排会更复杂

### 方案 C：高度模仿 Gashapon 的舞台式扭蛋页

把当前抽奖页做成更接近参考站的中心舞台型布局，结果球作为页面主角。

优点：

- 视觉冲击最强
- 最接近参考站体验

缺点：

- 会破坏当前项目整体统一性
- 会挤压奖品池、抽奖记录等已有内容
- 不符合“仍然放在一个扭蛋机里面”的要求

### 推荐结论

选择 **方案 B**。

这也是本次文档的默认方向：不改成全屏，不推翻当前结构，而是在现有扭蛋机卡片里完成一次“更灵动、更像真的扭蛋机升级”。

## 5. 页面结构原则

抽奖页整体仍保持以下结构：

1. 顶部标题与引导文案
2. 中部扭蛋机主体
3. 扭蛋币与抽奖按钮
4. 奖品池展示
5. 抽奖记录

但扭蛋机主体内部需要重新分层，改成以下视觉结构：

1. 玻璃罩层
2. 球体堆积层
3. 高光反射层
4. 机身装饰层
5. 旋钮交互层
6. 出蛋口层
7. 前景揭晓层

其中最关键的是第 7 层。它不是全屏弹窗，而是位于扭蛋机前方的“小型开奖舞台”，用于承接掉出的扭蛋球和开蛋动画。

## 6. 扭蛋机视觉改造

### 6.1 机身方向

当前机身的问题是圆润可爱，但轮廓比较平，没有“真实设备”的结构分区。

改造方向：

- 玻璃罩更像真实透明罩，厚度更明显
- 玻璃罩和机身连接处增加环形连接带
- 机身正面不再是一整块纯色面板，而是分成主体、旋钮区、出蛋区三个结构块
- 增加柔和但明确的高光、内阴影、反射边缘
- 保留爱心、蝴蝶结或丝带等少女元素，但更像“机器装饰”，不要像贴纸

建议机身气质：

- 奶油粉 + 淡樱花粉 + 香草黄
- 透明玻璃偏白粉光泽
- 金属/旋钮区域带一点温柔奶金色

### 6.2 机身细节

建议加入以下细节：

- 玻璃罩左上到中部的大面积弧形高光
- 玻璃罩边缘轻微描边，增强“罩子”存在感
- 机身中部增加品牌感装饰位，比如小爱心徽章或生日丝带
- 出蛋口更像真实凹槽，而不是单纯椭圆口
- 旋钮增加中心圆帽、外圈边框、转动刻线

目标不是写实电玩城，而是“少女风精致扭蛋机”。

## 7. 球体视觉改造

这是本次最重要的视觉升级点。

### 7.1 当前问题

现在的球体是统一大小、统一纯色、统一圆形阴影，缺少：

- 上下壳结构
- 接缝
- 半透明感
- 前后层次
- 被挤压堆叠的随机感

### 7.2 新球体方向

每个球体应更接近真实扭蛋球：

- 上半部分偏透明或半透明
- 下半部分是柔和实体色
- 中间有一圈清晰但柔和的拼接线
- 球体表面有两个高光点，而不是只有整体亮面
- 个别球体略大或略小，形成随机感
- 球体摆放角度和遮挡关系不完全一致

### 7.3 球体配色建议

建议使用 6 到 8 组球体配色轮换：

- 樱花粉球
- 奶油黄球
- 淡紫球
- 蜜桃粉球
- 草莓奶昔粉球
- 米白香草球
- 少量特殊款珠光球

特殊款球建议比例很低，只作为“隐藏大奖”或“幸运球”视觉暗示，不直接等于大奖结果。

### 7.4 球体摆放方式

机内球体不要规规矩矩排成几列，而要模拟堆积：

- 底部球更多
- 中上部球更稀疏
- 前景球略大、阴影更明显
- 后景球透明度略低
- 个别球部分被遮挡，避免“贴纸阵列感”

## 8. 交互流程改造

本次流程改造采用“五阶段开奖”。

### 阶段 1：待机态

页面初始状态下：

- 扭蛋机轻微上下浮动
- 机内球体慢速漂浮和轻碰撞
- 玻璃高光偶尔扫过
- 旋钮有轻微呼吸感
- 抽奖按钮保持可点击

目标：让用户感觉机器一直是“活的”。

### 阶段 2：启动抽奖

用户点击抽奖按钮后：

- 按钮先轻压再回弹
- 旋钮开始明显旋转
- 机身短促摇晃
- 玻璃罩内球体向下与向侧方翻动
- 出蛋口区域光晕增强
- 状态文案切换为“扭蛋正在掉出来……”

目标：制造“机器真的开始工作了”的反馈。

### 阶段 3：出蛋

机器不会直接给结果图标，而是先掉出一个完整扭蛋球：

- 一个被选中的扭蛋球从出蛋口滑出
- 扭蛋球落到扭蛋机前方的小舞台或小托盘
- 背景轻微压暗，但不进入全屏遮罩
- 页面提示用户“点一下打开扭蛋”

这是整次升级最关键的一步。

### 阶段 4：开蛋

用户点击这个球之后：

- 球先轻微弹跳确认点击
- 上下壳沿接缝分开
- 上半壳向上偏移并轻旋转
- 下半壳留在原位或稍微下沉
- 礼物图标、礼物名称或爱心粒子从球心升起
- 周围星光、彩屑、心形粒子扩散

这一阶段完成后，奖品才算真正揭晓。

### 阶段 5：结果确认

开蛋完成后：

- 奖品结果卡出现
- 奖品池里对应奖品高亮
- 抽奖记录更新
- 礼物口袋更新
- 扭蛋币扣减完成

这里的结果卡仍可使用现有 modal 体系，但视觉上应该更像“开奖结果卡”，而不是普通提示框。

## 9. 关键状态设计

为了让交互清晰，抽奖页需要显式区分以下状态：

- `idle`：待机
- `drawing`：机器抽奖中
- `capsule-ready`：球已掉出，等待点击打开
- `opening`：球体打开中
- `revealed`：奖品已揭晓

状态约束建议：

- `drawing` 和 `opening` 状态下，不允许再次点击抽奖
- `capsule-ready` 状态下，不允许切换标签页
- 奖品记录应在 `revealed` 后再写入，避免用户还没打开球，系统就先显示结果

## 10. 文案与微交互建议

建议把当前抽奖相关文案拆成阶段式文案：

- 待机：`准备好了就摇一摇，把今天的甜度拉满。`
- 抽奖中：`扭蛋机正在偷偷翻找今天的惊喜……`
- 出蛋后：`扭蛋掉出来啦，点开看看。`
- 开蛋中：`正在打开今天的小惊喜……`
- 已揭晓：`抽中了：xxx`

按钮文案也建议阶段化：

- 默认：`抽一发`
- 抽奖中：`扭蛋掉落中`
- 等待开蛋：`点扭蛋打开`
- 无币：`先去签到攒币`

## 11. 动效风格原则

动效不要做成赌场或电玩城风格，应保持柔和、甜、轻盈。

建议原则：

- 多用 `back.out`、`power2.out`、`elastic.out` 这类偏软的回弹
- 旋钮转动可以明确，但机身摇晃不要太暴力
- 球体弹跳可爱即可，不要高频抖动
- 粒子优先心形、星光、亮片，不做夸张烟花
- 所有动画以手机观看为优先，单次流程控制在 2.0 到 3.2 秒之间

## 12. 与当前实现的衔接建议

本次改造不需要推翻当前抽奖页，而是在现有三份核心文件上演进：

- `index.html`
- `styles.css`
- `script.js`

建议演进方向：

- `index.html`：补充“前景揭晓层”和“可点击扭蛋球”的结构
- `styles.css`：重做球体材质、机身结构细节、前景开奖舞台
- `script.js`：把当前“直接出奖品”改为“先出球，再开球，再写结果”

## 13. 推荐新增视觉组件

后续实现时，建议新增以下组件概念：

- `reveal-stage`：扭蛋机前方的揭晓舞台
- `reveal-capsule`：掉出的完整扭蛋球
- `capsule-shell-top`：扭蛋上壳
- `capsule-shell-bottom`：扭蛋下壳
- `capsule-core`：礼物内容核心层
- `open-tip`：点开提示
- `machine-reflection`：玻璃反光层
- `capsule-cluster`：机内球堆层

这些名字只是设计层命名，后面实现时可以再按代码结构调整。

## 14. 非目标

本次不做以下改造：

- 不改成全屏 Gashapon 页面
- 不加入复杂配置面板
- 不加入真实物理引擎
- 不加入音效依赖作为必需条件
- 不重做整个抽奖页其余区域的信息结构

## 15. 风险与注意事项

### 风险 1：动画太复杂导致手机卡顿

处理原则：

- 只使用 `transform` 和 `opacity` 做主要动画
- 球体数量控制在合理范围
- 待机动画强度低于开奖动画

### 风险 2：层级太多导致界面变乱

处理原则：

- 扭蛋机依然是视觉主角
- 奖品池和记录区域保持当前简洁卡片布局
- 揭晓舞台只在需要时出现，不常驻抢戏

### 风险 3：模仿过头，失去项目气质

处理原则：

- 借“节奏”不借“工具站外壳”
- 借“球体表现”不借“整页布局”
- 所有视觉语言都继续服务于生日、表白、可爱、少女感

## 16. 验收标准

改造完成后，抽奖页应满足以下标准：

1. 第一眼能明显看出这是一个更精致、更立体的少女风扭蛋机
2. 机内球体不再像规则圆点，而像真实扭蛋球堆
3. 抽奖结果不会直接弹出，而是先掉出一个完整球
4. 用户需要点击球才能正式打开惊喜
5. 整个流程在手机端顺畅、清楚、有期待感
6. 抽奖记录、奖品高亮、礼物口袋更新逻辑仍然正常

## 17. 后续实现建议

后续真正写代码时，推荐按下面顺序执行：

1. 先重做球体和机身样式
2. 再补“掉出完整扭蛋球”的结构
3. 再改 GSAP 时间线，拆成“抽奖 -> 出蛋 -> 开蛋 -> 揭晓”
4. 最后接入记录更新、状态切换和边界处理

---

这份文档的核心原则可以总结成一句话：

**保留现在的手机端卡片式扭蛋机框架，把参考站最有趣的“球体表现 + 开蛋仪式感”借进来，让抽奖从“按钮出结果”升级成“拿到扭蛋、亲手打开惊喜”。**
